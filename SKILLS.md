# Agent Skills

这个文件定义了 Agent 可以使用的技能。技能会根据上下文渐进式披露给 Agent。

## Skill: 文件操作

**描述**: 读取、写入、编辑文件的能力

**使用场景**:
- 需要创建或修改文件时
- 需要读取文件内容时
- 需要批量处理文件时

**可用工具**:
- `read_file`: 读取文件内容
- `write_file`: 创建或覆盖文件
- `edit_file`: 使用字符串替换编辑文件

**最佳实践**:
1. 编辑文件前先读取内容
2. 使用精确的字符串匹配进行编辑
3. 验证文件路径的有效性

**示例**:
```
1. 读取文件: read_file(path="/example.txt")
2. 编辑文件: edit_file(path="/example.txt", old_string="old", new_string="new")
3. 写入文件: write_file(path="/new.txt", content="Hello World")
```

## Skill: 文件搜索

**描述**: 搜索和查找文件的能力

**使用场景**:
- 需要查找特定文件时
- 需要搜索文件内容时
- 需要列出目录内容时

**可用工具**:
- `ls`: 列出目录内容
- `grep`: 在文件中搜索模式
- `glob`: 查找匹配模式的文件

**最佳实践**:
1. 使用 glob 查找文件名模式
2. 使用 grep 搜索文件内容
3. 使用 ls 浏览目录结构

**示例**:
```
1. 列出目录: ls(path="/")
2. 搜索内容: grep(pattern="TODO", path="/")
3. 查找文件: glob(pattern="*.go", path="/")
```

## Skill: 任务规划

**描述**: 创建和管理任务列表的能力

**使用场景**:
- 需要分解复杂任务时
- 需要跟踪任务进度时
- 需要规划工作流程时

**可用工具**:
- `write_todos`: 创建或更新 Todo 列表

**最佳实践**:
1. 将大任务分解为小任务
2. 为每个任务设置清晰的状态
3. 及时更新任务进度

**示例**:
```json
{
  "todos": [
    {
      "id": "1",
      "title": "创建项目结构",
      "status": "completed",
      "description": "创建基本的目录结构"
    },
    {
      "id": "2",
      "title": "实现核心功能",
      "status": "in_progress",
      "description": "实现主要的业务逻辑"
    }
  ]
}
```

## Skill: 代码开发

**描述**: 编写和修改代码的能力

**使用场景**:
- 需要实现新功能时
- 需要修复 bug 时
- 需要重构代码时

**最佳实践**:
1. 遵循项目的编码规范
2. 编写清晰的代码注释
3. 保持函数小而专注
4. 先编写测试再实现功能

**相关技能**: 文件操作、文件搜索

## Skill: 测试编写

**描述**: 编写和运行测试的能力

**使用场景**:
- 实现新功能后需要测试时
- 修复 bug 需要验证时
- 重构代码需要保证正确性时

**最佳实践**:
1. 为每个功能编写测试
2. 测试边界情况和错误处理
3. 保持测试简单和独立
4. 使用有意义的测试名称

**相关技能**: 代码开发、文件操作
